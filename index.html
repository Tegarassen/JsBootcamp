What is JavaScript?

JavaScript is a scripting language that runs in the browser (and on servers via Node.js). It is case-sensitive and dynamically typed — you don't declare variable types.

What JS can do

  • Add interactive behaviour to web pages
  • Build web & mobile apps (React, React Native)
  • Run on servers with Node.js
  • Power browser-based games

Module 1
Variables & Scope
Understanding how JS stores data and where that data can be accessed.

Declaring variables: var, let, const

var Avoid

  • Function scoped
  • Can be re-declared
  • Leaks to parent scope
  • Causes confusing bugs

let Use when reassigning

  • Block scoped ({})
  • Can be reassigned
  • Cannot be re-declared
  • Safer than var

const Default choice

  • Block scoped ({})
  • Cannot be reassigned
  • Makes intent clear
  • Use for everything else
💡 Rule of thumb: always start with const. Switch to let only if you need to reassign. Never use var.

Scope visualised

Scope determines where a variable can be accessed. var is function-scoped; let and const are block-scoped.

scope-demo.js
function testVar() { if (true) { var name = 'admin'; // leaks out of the block } console.log(name); // "admin" — accessible! } function testLet() { if (true) { let name = 'admin'; } try { console.log(name); } catch(e) { console.log('ReferenceError: name is not defined'); } } testVar(); testLet();

The var loop trap

A classic gotcha: using var in a for loop with setTimeout leaks the same variable across all iterations.

var-trap.js
// With var — all callbacks share the SAME i const varResults = []; for (var i = 0; i < 3; i++) { varResults.push(i); } console.log("var results:", varResults); // [0,1,2] — but if async, always 3 // With let — each iteration gets its OWN scope const letResults = []; for (let j = 0; j < 3; j++) { letResults.push(j); } console.log("let results:", letResults); // [0,1,2] — always correct

TypeError: Assignment to constant variable

undefined
✅ Correct! const prevents reassignment. You'll get a TypeError at runtime.
Module 2
Data Types
JavaScript is dynamically typed — you don't declare types, but they still matter.

undefined vs null vs empty string

❓

undefined

Variable declared but not given a value yet.

let a; a === undefined

When you see it: uninitialized vars, missing params, missing object props

🚫

null

Intentionally set to "no value". Used with objects.

let a = null; a === null

When you see it: resetting object properties, APIs returning "nothing"

📭

Empty String ""

A string with no characters. Still a valid string.

let a = ''; a.length === 0

When you see it: form fields, string ops that produce no content

typeof & comparisons

types.js
let a; console.log(typeof a); // "undefined" console.log(a === undefined); // true let b = null; console.log(typeof b); // "object" (JS quirk!) console.log(b === null); // true let c = ''; console.log(typeof c); // "string" console.log(c.length === 0); // true
⚠️ typeof null returns "object" — this is a famous JavaScript bug kept for backwards compatibility. Always use === null to check for null.

Equality: == vs ===

JavaScript has two equality operators. === (strict) checks both value AND type. == (loose) coerces types first — avoid it.

equality.js
console.log(1 == '1'); // true — coerces string to number console.log(1 === '1'); // false — different types console.log(1 === 1); // true console.log(1 == true); // true — coerces boolean console.log(1 === true); // false — different types
💡 Always use ===. The only time == is useful is checking for both null and undefined at once: if (x == null)

Quick quiz

What does typeof null return?

"null"
"object"
"undefined"
"number"
✅ Correct! typeof null === "object" is a long-standing JS quirk. Don't use typeof to check for null — use === null instead.
Module 3
Strings
From basic concatenation to ES6 template literals and built-in string methods.

Template literals (ES6)

Use backticks ` instead of quotes. Embed any expression with ${ }. Multi-line strings just work.

template-literals.js
const name = 'Kevan'; const price = 19.99; const tax = 0.07; // Old way console.log('Hello, ' + name + '!'); // Template literal console.log(`Hello, ${name}!`); // Expression inside ${ } console.log(`Total: $${(price * (1 + tax)).toFixed(2)}`); // Multi-line console.log(`Line 1 Line 2`);

Useful string methods

.includes(str)

Returns true if string contains the substring

.startsWith(str)

Returns true if string starts with value

.endsWith(str)

Returns true if string ends with value

.repeat(n)

Returns string repeated n times

.toUpperCase()

Returns uppercased copy

.trim()

Removes whitespace from both ends

.split(sep)

Splits into an array by separator

.slice(start, end)

Extracts part of a string

string-methods.js
const email = 'keshav@spoonconsulting.com'; console.log(email.includes('spoon')); // true console.log(email.endsWith('.com')); // true console.log(email.toUpperCase()); // KESHAV@SPOONCONSULTING.COM console.log('ha'.repeat(3)); // hahaha console.log(email.split('@')); // ["keshav", "spoonconsulting.com"]

Arrays
Ordered collections, destructuring, and the powerful array methods every JS developer must know.

Array basics & destructuring

arrays.js
const details = ['Swankii', 'Pakii', 'toke.com']; // Access by index console.log(details[0]); // Swankii // Destructuring — unpack into named variables const [first, last, website] = details; console.log(first); // Swankii console.log(last); // Pakii console.log(website); // toke.com

Key array methods

These methods are the backbone of modern JavaScript. Each takes a callback function.

array-methods.js
const numbers = [1, 2, 3, 4, 5, 6]; // .map() — transform each element, returns new array const doubled = numbers.map(n => n * 2); console.log('map:', doubled); // .filter() — keep elements that pass the test const evens = numbers.filter(n => n % 2 === 0); console.log('filter:', evens); // .reduce() — accumulate into a single value const sum = numbers.reduce((acc, n) => acc + n, 0); console.log('reduce sum:', sum); // .find() / .findIndex() console.log('find >4:', numbers.find(n => n > 4)); console.log('findIndex >4:', numbers.findIndex(n => n > 4));

Module 5
Objects
Key-value pairs, destructuring, renaming, defaults, and enhanced object literals.

Object basics & destructuring

objects.js
const person = { firstName: 'Kevan', lastName: 'Moothien', location: { city: 'Las Vegas', state: 'Nevada' } }; // Basic destructuring const { firstName, lastName } = person; const { city, state } = person.location; console.log(firstName, lastName); // Kevan Moothien console.log(city, state); // Las Vegas Nevada // Rename while destructuring const { firstName: fName, lastName: lName } = person; console.log(fName, lName); // Kevan Moothien // Default values in destructuring const settings = { color: 'white', height: 500 }; const { width = 200, height = 200, color = 'black' } = settings; console.log(width, height, color); // 200 500 white

Enhanced object literals (ES6)

Old way

const person = { first: first, last: last, greet: function() { console.log('hi'); } };

ES6 shorthand

const person = { first, // same as first: first last, greet() { // shorthand method console.log('hi'); } };
💡 When the variable name matches the property name, you can just write the variable name once. This is one of the most common ES6 patterns you'll see in React and Node.js codebases.

Module 6
Control Structures
Making decisions in your code with if/else and switch statements.

if / else if / else

if-else.js
const time = 24; let greeting; if (time < 10) { greeting = "Good morning"; } else if (time < 20) { greeting = "Good day"; } else { greeting = "Good evening"; } console.log(greeting); // Good evening

switch statement

Use switch when comparing a single value against many fixed options. Always include break and a default case.

switch.js
const day = 3; // 0=Sun, 1=Mon, ..., 6=Sat let text; switch (day) { case 6: text = "Today is Saturday"; break; case 0: text = "Today is Sunday"; break; default: text = "Looking forward to the weekend!"; } console.log(text);
⚠️ Forgetting break causes "fall-through" — execution continues into the next case. This is rarely what you want.

Quick quiz

When should you prefer switch over if/else if?

When you need to compare ranges (e.g. x < 10)
When comparing one variable against many exact values
switch is always better than if/else
When you have only 2 conditions
✅ Correct! Switch shines when matching a single value against a set of known constants. For range checks, use if/else.
Module 7
Loops
Four ways to repeat code: for, for...of, for...in, while, and do...while.

Loop types at a glance

for

Classic index-based. Use when you need the index or control over step size.

for...of

Iterates values of an array. Clean and readable — the modern default.

for...in

Iterates keys of an object. Don't use on arrays.

while / do...while

Loop while a condition is true. do...while always runs at least once.

All four in action

loops.js
const cars = ['Honda', 'Toyota', 'BMW']; // 1. Classic for for (let i = 0; i < cars.length; i++) { console.log('for:', cars[i]); } // 2. for...of (arrays) for (const car of cars) { console.log('for...of:', car); } // 3. for...in (objects) const users = { user1: 'Tom', user2: 'Paul' }; for (const key in users) { console.log('for...in:', users[key]); } // 4. while let i = 0; while (i < cars.length) { console.log('while:', cars[i++]); }

Functions & ES6 Features
Arrow functions, default arguments, and the ES6 features that changed everything.

Arrow functions

Arrow functions are a concise way to write functions introduced in ES6. They also inherit this from the surrounding scope (lexical this).

arrow-functions.js
// Traditional const add1 = function(a, b) { return a + b; }; // Arrow function const add2 = (a, b) => { return a + b; }; // Implicit return (no braces needed for single expression) const add3 = (a, b) => a + b; console.log(add1(2, 3)); // 5 console.log(add2(2, 3)); // 5 console.log(add3(2, 3)); // 5 // Arrow function with default parameters const greet = (name = 'World') => `Hello, ${name}!`; console.log(greet()); // Hello, World! console.log(greet('Keshav')); // Hello, Keshav!

Default parameters (ES6)

Old way

function calcTotal(sub, tax, ship) { if (tax === undefined) tax = 0.07; if (ship === undefined) ship = 10; return sub + ship + (sub * tax); }

ES6 defaults

function calcTotal( sub, tax = 0.07, ship = 10 ) { return sub + ship + (sub * tax); }

Module 9
Spread & Rest Operators
The ... operator — two names, two uses, one powerful syntax.

Spread operator ...

Spread expands an array or object into individual elements.

spread.js
// Combine arrays const a = ['one', 'two']; const b = ['three', 'four']; const combined = [...a, ...b]; console.log(combined); // Insert at any position const inserted = ['zero', ...a, 'two-and-half', ...b]; console.log(inserted); // Copy an array (no mutation!) const copy = [...a]; copy.push('added'); console.log('original:', a); console.log('copy:', copy); // Spread into function arguments const nums = [5, 2, 9, 1]; console.log('max:', Math.max(...nums));

Rest operator ...

Rest collects remaining arguments into an array. Used in function parameters and destructuring.

rest.js
// Rest in function params function doMath(operator, ...numbers) { console.log('operator:', operator); console.log('numbers:', numbers); if (operator === 'add') { return numbers.reduce((sum, n) => sum + n, 0); } } console.log(doMath('add', 1, 2, 3, 4)); // Rest in destructuring const [first, second, ...rest] = ['Kevin', 'Bobby', 'Nicole', 'Naomi']; console.log(first); // Kevin console.log(second); // Bobby console.log(rest); // ["Nicole", "Naomi"]
ℹ️ Same syntax, different direction: Spread expands. Rest collects. The key is position: spread appears in a function call or array literal; rest appears in a function definition or destructuring pattern.

Promises & Async/Await
Handling asynchronous operations — the modern way to work with APIs and timers.

What is a Promise?

A Promise represents a value that may be available now, later, or never. It can be in one of three states:

⏳ Pending
Initial state
→
✅ Fulfilled
❌ Rejected
→
.then(data => ...)
.catch(err => ...)

Creating & chaining Promises

promises.js
// Create a promise that resolves const p1 = new Promise((resolve, reject) => { resolve('Learn JavaScript Deeply!'); }); p1.then(data => console.log('resolved:', data)); // Create a promise that rejects const p2 = new Promise((resolve, reject) => { reject(new Error('Something went wrong!')); }); p2 .then(data => console.log(data)) .catch(err => console.log('caught:', err.message));

async / await

async/await is syntactic sugar over Promises — it makes async code look and read like synchronous code.

async-await.js
// Helper: returns a promise after a delay const wait = (ms, val) => new Promise(res => setTimeout(() => res(val), ms)); // async function — always returns a Promise const runExample = async () => { console.log('Starting...'); // await pauses execution until the Promise resolves const result = await wait(100, 'Step 1 done'); console.log(result); const result2 = await wait(100, 'Step 2 done'); console.log(result2); console.log('All done!'); }; runExample();
💡 Always wrap await in a try/catch in production code to handle rejected promises gracefully.

Quick quiz

✅ Correct! await must be inside an async function (except at the top level of ES modules).

Object-oriented programming in ES6 and how to organise code across files.

Classes & Inheritance

Animal
constructor(name)
speak() → logs noise
↓
extends (inherits)
↓
Dog
constructor(name, breed)
set year(year)
get age() → calculates
speak() → overrides Animal
classes.js
class Animal { constructor(name) { this.name = name; } speak() { console.log(`${this.name} makes a noise.`); } } class Dog extends Animal { constructor(name, breed) { super(name); // calls Animal's constructor this.breed = breed; this.birthYear = null; } set year(y) { this.birthYear = y; } get age() { if (!this.birthYear) return null; return new Date().getFullYear() - this.birthYear; } speak() { console.log(`${this.name} barks!`); } } const puppy = new Dog('Spot', 'Doberman'); puppy.year = 2020; console.log(`${puppy.name}'s age: ${puppy.age}`); puppy.speak();

ES6 Modules

Modules let you split code across files. Two styles: named exports (multiple per file) and default exports (one per file).

Named exports

// mathlib.js export function square(x) { return x * x; } export function add(x, y) { return x + y; } // main.js import { square, add } from './mathlib'; console.log(square(9)); // 81

Default export

// foo.js export default function() { console.log('Foo!'); } // main.js import foo from './foo'; foo(); // Foo!
ℹ️ A file can have many named exports but only one default export. Named exports must be imported with {}; default exports are imported with any name you choose.

What is ES6?

ES stands for ECMAScript — the standard JavaScript is based on. ES6 (2015) was a massive update that introduced most of the features we've covered: let/const, arrow functions, template literals, destructuring, classes, modules, promises, and more.

ES6 / 2015 let, const, arrow functions, classes, modules, promises, template literals, destructuring, spread/rest, default params

ES7–ES15 async/await (ES8), optional chaining, nullish coalescing, Array.at(), Object.hasOwn(), and more refinements each year
